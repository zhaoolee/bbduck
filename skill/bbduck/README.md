# bbduck

一个面向 Hermes 的本地图片压缩 skill。

核心卖点只有一句话：
- 优先做“视觉无损压缩”——先尽量保证看起来没变，再把体积压下来。

默认连接本机 Docker 服务：

```bash
docker run -d -p 28642:8000 zhaoolee/bbduck:latest
```

默认能力：
- 支持 jpg / png / webp / gif 四种类型图片压缩
- 支持单张图片压缩调用
- 支持读取后端流式压缩日志
- 日志包含每一步的 `spend_time_ms`
- 默认压缩模式与网页一致：`visual-lossless`
- 支持批量下载压缩结果 ZIP

为什么这个 skill 值得装：
- 不是单纯追求极限压缩率，而是更贴近真实用户需求
- 特别适合“图片想变小，但不想一眼看出画质被动过”的场景
- 压缩过程可观测，适合 UI 展示、自动化调用和调试
- 与本地 Docker 服务配合，开箱即用

适合的典型需求：
- “帮我压一张图，但尽量别看出变化”
- “把压缩过程的每一步日志打出来”
- “压完后把结果一起打包下载”

安装方式：
```bash
cp -a skill/bbduck ~/.hermes/skills/
```

安装后，这个 skill 会把 bbduck 服务理解为：
- 一个默认走 `visual-lossless` 的图片压缩接口
- 一个支持流式日志的单图压缩工具
- 一个可用于批量下载压缩结果 ZIP 的辅助接口
